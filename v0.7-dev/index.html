TGUI v0.7 development version

This page is about the current version in development. You will find links to tutorials and documentation here, but keep in mind that everything can still change.

The expected release date for the final 0.7 version (or at least a release candidate) is set to September 2015.

Minimum requirements

The requirements for v0.7 differ from the ones in v0.6.

First of all, you need sfml 2.2, it will NOT compile with sfml 2.1 or sfml 2.0.

Secondly, these are the new minimal supported compilers:

  • VC++12 (VS2013)
  • gcc 4.7
  • clang 3.2

Features

Here is a list of some important features that are already implemented and available.

  • Experimental Android and iOS support.
  • The layout system. You can now give widgets a position or size relative to another one. The value will automatically be updated when the bound widget changes.
  • There is a new callback system
  • Exceptions are now used for error handing
  • Widgets no longer need textures. There is also a simple colored version for each widget. There is even a built-in theme that is used when you don't provide any theme file.
  • Label and TextBox were rewritten. The biggest addition is probably that they now have word wrap.
  • Many small improvements to all widgets. Stuff like allowing the text color to change on mouse hover in button and checkbox.
  • TGUI now has its own view. You no longer have to change the view every frame between sfml and tgui rendering, while not forgetting to have the right view set during event handling.
  • ...
